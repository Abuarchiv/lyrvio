# Lyrvio — Accessibility-Spec (WCAG 2.1 AA)

## Zielkonformität

**Standard:** WCAG 2.1 Level AA  
**Scope:** Onboarding-Flow, Dashboard, Extension-Popup  
**Testing-Tools:** axe DevTools, Chrome Accessibility Tree, VoiceOver (macOS), NVDA (Windows)

---

## 1. Tastatur-Navigation

### Globale Regeln
- Jedes interaktive Element ist via `Tab` erreichbar
- Tab-Reihenfolge folgt visueller DOM-Reihenfolge (kein `tabindex > 0`)
- Skip-Link: `<a href="#main-content" class="sr-only focus:not-sr-only ...">Zum Inhalt springen</a>` im Layout-Header
- Kein Tastaturfallen (Keyboard Traps) außer in Modalen

### Onboarding-Navigation
- `Tab` / `Shift+Tab`: zwischen Inputs, Labels, Buttons
- `Enter` / `Space`: Buttons und Checkboxen aktivieren
- `Arrow Keys`: Radio-Groups navigieren (Step 4 Schufa-Score)
- Step 5 DistrictMap: `Tab` zwischen Bezirk-Punkten, `Space` zum Auswählen
- Step 6 Range-Slider: `Arrow Keys` für +/- Werte, `Shift+Arrow` für ×10

### Fokus-Styles
```css
/* Konsistenter Focus-Ring für alle Elemente */
:focus-visible {
  outline: 2px solid rgb(99, 102, 241); /* indigo-500 */
  outline-offset: 2px;
  border-radius: 4px;
}
```
- Kein `outline: none` ohne Alternative
- Focus-Ring kontrastreich auf dunklem Hintergrund: #818CF8 auf #0a0a0f = 6.3:1 (AA+)

## 2. Screen-Reader-Labels

### Globale Patterns

```tsx
// Buttons mit Icon only → aria-label
<button aria-label="Bezirk Neukölln entfernen">
  <X className="w-3 h-3" aria-hidden="true" />
</button>

// Icons dekorativ → aria-hidden
<CheckCircle2 className="w-5 h-5" aria-hidden="true" />

// Error-Messages → aria-describedby + role="alert"
<Input aria-invalid={!!error} aria-describedby="email-error" />
<p id="email-error" role="alert">{error.message}</p>

// Progress-Bar
<div role="progressbar" aria-valuenow={3} aria-valuemin={1} aria-valuemax={6} aria-label="Schritt 3 von 6" />

// Radio-Gruppen
<div role="group" aria-label="Schufa-Score auswählen">
  <label><input type="radio" ... /> Sehr gut</label>
  ...
</div>

// Formulare
<form aria-label="Persönliche Daten eingeben">
  <fieldset>
    <legend className="sr-only">Persönliche Informationen</legend>
    ...
  </fieldset>
</form>
```

### Spezifische Komponenten

**StepProgress:**
- `role="progressbar"` + `aria-valuenow/min/max`
- `aria-current="step"` auf aktivem Step-Dot
- Desktop-Nav: `<nav aria-label="Onboarding Fortschritt"><ol>...</ol></nav>`

**DistrictMap:**
- Container: `role="group" aria-label="Bezirkskarte Berlin"`
- Jeden Bezirk-Button: `aria-pressed={isSelected}` + `aria-label="Bezirk Neukölln auswählen/abwählen"`
- Chips: `<div aria-label="Ausgewählte Bezirke">...</div>`
- Remove-Buttons: `aria-label="Neukölln entfernen"`

**MappeUploader:**
- Drop-Zone: `role="region" aria-label="Mietmappe hochladen"`
- File-Input: `aria-label="Mietmappe Datei auswählen"`
- Progress-Bar: `role="progressbar" aria-valuenow aria-valuemin aria-valuemax`
- Status-Meldungen: `role="status" aria-live="polite"`
- Error-Meldungen: `role="alert"`

**StepNavigation:**
- Zurück-Button: `aria-label="Zurück zum vorherigen Schritt"`
- Weiter-Button: `aria-live="polite"` (ändert sich bei isSubmitting)

## 4. Formulare & Fehlerbehandlung

### Error-Announcement
- React Hook Form Fehler → `role="alert"` auf Error-Paragraph
- Beim Submit: Focus springt auf erstes fehlerhaftes Feld
- `aria-invalid="true"` auf fehlerhaften Inputs
- Alle Error-IDs eindeutig per Step: `email-error`, `vollmacht-error`, etc.

### Required Fields
- Alle Pflichtfelder: `aria-required="true"`
- Keine rein visuellen Asterisk-Markierungen ohne Screenreader-Text

### Hints
- Separate `<p id="X-hint">` mit `aria-describedby` statt Placeholder-Abuse

---

## 5. Bilder & Icons

- Alle Lucide-Icons: `aria-hidden="true"` (dekorativ)
- Illustrationen: `alt="..."` oder `aria-hidden="true"` + Textäquivalent
- PDF-Upload-Vorschau: `alt="Hochgeladene Datei: {filename}"`

---

## 6. Bewegung & Animation

- Alle CSS-Animationen respektieren `prefers-reduced-motion`:

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

- Kein Auto-Play-Video
- Loader-Spinner: `aria-live="polite"` statt Blink

---

## 7. Modals & Dialoge (zukünftig)

- `role="dialog"` + `aria-modal="true"` + `aria-labelledby`
- Focus-Trap innerhalb Modal
- `Esc` schließt Modal
- Beim Schließen: Focus zurück zum Trigger-Element
